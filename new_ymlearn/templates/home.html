Connecting to the database and accessing code base.
Before you can can make sense of the completed tasks, you need to first connect to the database and you must be able connect to the code base.

Connecting to the database

  • Step 1 - Download DBbeaver
  • Step 2 - Once DBbeaver has been downloaded, create a database connection. Select PostgreSQL
  • Step 3 - On the main tab, enter connection details. Request this details from Yonis Mohamoud
  • Step 4 - You should be able to connect to the database
  • Step 5 - Familiarise yourself with the database schema . A screenshot of the schema will be provided here once ready.

Accessing the codebase

  • Step 1 - Visit the GitHub repository here. The repository has been deliberately left on public so heroku can access it. Once a push is made to the main branch, the app on heroku is automatically update with the latest code changes.
  • Step 2 - To set the code locally, you need to install the packages in the requirements file. Simply navigate to the directory containing the requirements.txt file in command prompt or terminal and run the command pip install -r requirements.txt
  • Step 3 - To launch the website on your local machine, navigate to the directory containing the file server.py and run the command python server.py
  • Step 4 - The login functionality has not been developed yet, so for the time being, click on login by pass to skip past this page.
Completed/pending tasks
The website is fully dynamic. A user should be able to easily create a new course by simply inserting values to various database tables. Below are the tasks that have been completed (see text in green) and tasks that are pending (see text in red):
  • Step 1 - Insert new course into table course. An id, course_name, and flag that determines whether the course is available needs to be provided. For course_available column, if select No, you won't see the topic in the qualification dropdown
  • Step 2 - Insert a topic(s). There is a Foreign key to the course table. You need to link the new topic(s) back to the course. This will help your user see what topics are linked to the course i.e. Number, Geometry etc..
  • Step 3 - Insert a sub topic(s). There is a Foreign key to the topic table. You need to link the sub topic back to the topic. This is how you are able to see a massive list of under Number, Algebra etc...
  • Step 4 - Insert learning objective(s). There is a foreign key to the sub topic table. You will need to link leaning objective back to the subtopic. For topics that are available, there is an option to select information icon to see the learning objectives. You can also see learning objectives by selecting Go to topic
  • Step 5 - Insert Videos and files to a specific sub_topic
  • Step 6 - Create the login and log off functionality
  • Step 7 - Insert practice questions where students can answer questions and get feedback on performance
  • Step 8 - Insert exam style questions where students can answer questions and get feedback on performance
  • Step 9 - Record student's First test and last test result on sub topic page. All student's test results to be stored in database table
  • Step 10 - Create a dashboard page where admin can access student's result
  • Step 11 - Create accounts specific to teachers
  • Step 12 - Students can be assigned to a teacher
  • Step 13 - Create a dashboard page for teachers so they can access their student's details
  • Step 14 - Enable password reset so students can do it themselves or admin can do it for them.
